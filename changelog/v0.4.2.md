# Razy v0.4.2

**Build 212** | November 2023

Restructured the module package format, introduced `ModuleInfo`, and added the `commit` terminal command for module versioning.

---

## Build 212 (2023-11-24)

### Breaking Changes

- Changed module file structure — all files under the module folder must be moved into a `default` folder
- Rewritten module package structure — all module locations must follow the `vendor/module` rule
- Removed `assets` parameter in `package.php` — replaced by `webassets` folder under the module root
- Removed `unpackasset` command — replaced by `rewrite` command for updating `.htaccess` rewrite rules
- Removed `Distributor::unpackAllAsset()`

### Additions

- Added `terminal::commit` command for committing a module to a specified version
- Distributor added `identifier` for versioning purposes
- Extracted `Module` general functions into `ModuleInfo` — allows obtaining module details directly
- Added `Distributor::GetDistributorByName()` to get a distribution by name
- Added `getWeekdayDiff()` to compare two dates and return the weekday difference

### Fixes

- Fixed `is_fqdn` cannot detect the correct IP format
- Fixed `Distributor` validating unloaded modules
- Fixed `Statement::getSyntax()` generating incorrect `LIMIT` statement

### Changes

- Revised `.htaccess` template
