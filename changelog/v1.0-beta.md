## Composer-Compatible Package Management

New documentation added to wiki and documentation site covering:

- **Module prerequisite declaration** — `prerequisite` key in `package.php` for declaring Composer package dependencies (distinct from `require` for Razy module dependencies)
- **Compose workflow** — `php Razy.phar compose <distCode>` collects all prerequisites, resolves versions, downloads, and extracts packages
- **Per-distributor isolation** — packages extract into `autoload/{distributor_code}/` with PSR-4/PSR-0 mapping; `autoload/lock.json` tracks versions scoped by distributor

---

## Build System

- `build.php` updated to accept CLI path arguments for PHAR export: `php build.php [path1] [path2] ...`
- Removed hardcoded copy path
- `chmod` skipped on Windows (`PHP_OS_FAMILY` guard)

---

## Standalone Mode

- Extracted `Standalone` class from `Distributor` into `Razy\Distributor\Standalone`
- Introduced `DistributorInterface` contract for type-safe distributor access
- Updated all consumer code to use interface-based type hints
