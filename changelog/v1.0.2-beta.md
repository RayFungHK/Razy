## Security Enhancements

### Critical Fixes

| # | Vulnerability | File(s) | Fix |
|---|---|---|---|
| 1 | **SQL injection via dynamic table/view names** | `Database.php`, `Statement.php` | Regex validation (`/^[a-zA-Z_]\w*$/`) on all table and view name parameters before interpolation |
| 2 | **Unsafe object deserialization in cache** | `Cache/FileAdapter.php` | `unserialize()` called with `['allowed_classes' => false]` to block object injection |
| 3 | **SSRF via unrestricted cURL protocols** | `Http/HttpClient.php` | `CURLOPT_PROTOCOLS` restricted to `CURLPROTO_HTTP \| CURLPROTO_HTTPS` |
| 4 | **Secret exfiltration via Google Charts QR API** | `Authenticator.php` | Removed external API call; `generateProvisioningUri()` / `generateHotpProvisioningUri()` now return `otpauth://` URI directly |

### High-Severity Fixes

| # | Vulnerability | File(s) | Fix |
|---|---|---|---|
| 5 | **Dashboard API unauthenticated access** | `dashboard/router.php` | Token-based authentication via `RAZY_DASHBOARD_TOKEN` env var with `hash_equals()` comparison |
| 6 | **Path traversal in dashboard static assets** | `dashboard/router.php` | Directory traversal sequence rejection (`..`, encoded variants) on asset and config paths |
| 7 | **CLI proxy command injection** | `dashboard/router.php` | Allowlisted CLI commands; strict argument validation in CLI proxy endpoint |
| 8 | **Null byte path truncation** | `FileReader.php` | `\0` byte detection and rejection in all file path methods with `FileException` |
| 9 | **CORS origin regex bypass** | `XHR.php` | Anchored regex pattern `/^https?:\/\/(?:[\w\-]+\.)*[\w\-]+(?::\d+)?$/` on all origin entries |
| 10 | **OAuth2 state parameter forgery** | `OAuth2.php` | New `verifyState()` method for CSRF protection; HTTPS enforcement warnings on URL setters |

### Medium-Severity Fixes

| # | Vulnerability | File(s) | Fix |
|---|---|---|---|
| 11 | **Process injection via package names** | `pkg.inc.php` | Backslash sanitisation in `pkg stop` ŌĆö strip `\` from package identifiers before shell execution |
| 12 | **WMIC command injection on Windows** | `PackageRunner.php` | `basename()` + alphanumeric-only regex filter before WMIC `CommandLine like` query |
| 13 | **PHP binary injection** | `PackageRunner.php` | `escapeshellarg()` wrapping `PHP_BINARY` and all phar target paths in process spawn commands |
| 14 | **Error message information leakage** | `FileReader.php` | Sanitised exception messages ŌĆö no longer expose full filesystem paths |

## Package System

### New Classes

| Class | Purpose |
|---|---|
| `Razy\Package\PackageManifest` | Parses and validates `package.json` manifests inside `.phar` archives (name, version, entry point, dependencies, serve mode config) |
| `Razy\Package\PackageRegistry` | Manages installed packages ŌĆö catalogue, lookup, version resolution, dependency graph |
| `Razy\Package\PackageRunner` | Runs `.phar` packages (one-shot and serve-mode daemons), handles process lifecycle (spawn, PID tracking, stop), Composer autoload bootstrapping |

### `pkg` CLI Command

Unified package management via `php Razy.phar pkg <sub-command>`:

| Sub-command | Description |
|---|---|
| `pkg list` | List all installed packages with version, type, and status |
| `pkg info <package>` | Show detailed package metadata (dependencies, entry point, config) |
| `pkg install <package>[@version]` | Install package from configured repository |
| `pkg publish [options]` | Publish packages to GitHub-based repository |
| `pkg stop <package> \| --all` | Stop a running serve-mode package daemon (or all) |
| `pkg run <package>` | Run a package in foreground mode |

### `pkg publish` Flags

| Flag | Description |
|---|---|
| `--scan` | Scan `packages/` directory for publishable modules |
| `--push` | Push scanned packages to the configured GitHub repository |
| `--force` | Overwrite existing versions in the repository |
| `--cleanup` | Remove orphaned releases/tags after publishing |
| `--verbose` | Detailed output during publish operations |
| `--dry-run` | Preview publish operations without making changes |

### Repository & Publishing

- Added `type` field to `RepositoryManager` for package type detection (module, application, library)
- `publish.inc.php` refactored ŌĆö standalone CLI command closure removed; retained as helper function library (`normalizeVersion`, GitHub API functions, `packModule`) loaded on demand by `pkg publish`
- Credential configuration moved from `publish.inc.php` inline to `publish.json` (excluded from phar packing)

## Dashboard Package

A built-in web dashboard for real-time framework monitoring, shipped as a standalone `.phar` package.

### UI Tabs

| Tab | Content |
|---|---|
| **Overview** | Health status, PHP version, framework version, uptime |
| **System** | Loaded extensions, OPcache status, memory usage, ini settings |
| **Packages** | Installed packages table with version, type, status |
| **Sites** | Configured sites/distributors with domain mapping |
| **Cache** | Cache adapter status, hit/miss statistics, flush/clear/warm actions |
| **CLI** | Interactive terminal with command proxy to `Razy.phar` CLI |

### API Endpoints

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/health` | GET | Framework health check |
| `/api/system` | GET | PHP and system information |
| `/api/packages` | GET | Installed package listing |
| `/api/sites` | GET | Site/distributor configuration |
| `/api/cache/status` | GET | Cache adapter status |
| `/api/cache/stats` | GET | Cache hit/miss statistics |
| `/api/cache/flush` | POST | Flush cache |
| `/api/cache/clear` | POST | Clear cache |
| `/api/cache/warm` | POST | Warm cache |
| `/api/cli` | POST | CLI command proxy |
| `/api/inspect` | GET | Distributor inspection |
| `/api/routes` | GET | Route table dump |
| `/api/validate` | GET | Configuration validation |

### Dashboard Security

- Token-based authentication via `RAZY_DASHBOARD_TOKEN` environment variable
- Timing-safe comparison with `hash_equals()`
- Path traversal protection on all file-serving endpoints
- CLI proxy restricted to allowlisted commands
- UTF-8 sanitisation on CLI subprocess output (Windows codepage compatibility)
- `JSON_INVALID_UTF8_SUBSTITUTE` flag for resilient JSON encoding
- Client-side error handling with descriptive fallback messages

## Bug Fixes

| Issue | File | Fix |
|---|---|---|
| Asset / `getAssetPath()` missing subdirectory prefix (e.g. `https://localhost/webassets/ŌĆ”` instead of `https://localhost/abc/webassets/ŌĆ”`) | `bootstrap.inc.php` | `RELATIVE_ROOT` now requires `DOCUMENT_ROOT` as a strict path prefix of `SYSTEM_ROOT` (avoids false `strpos` matches); when that cannot be computed, falls back to `dirname(SCRIPT_NAME)` so subdirectory installs resolve the correct URL base |
| Dashboard "Failed to execute 'json' on 'Response'" error | `dashboard/router.php`, `index.html` | Non-UTF-8 bytes from Windows CLI subprocess caused `json_encode()` to return `false`, sending empty body. Added `mb_check_encoding()` / `mb_convert_encoding()` sanitisation and `JSON_INVALID_UTF8_SUBSTITUTE` flag |
| `--scan` flag not recognised by `pkg publish` | `pkg.inc.php` | `main.php` strips one leading dash, so `--scan` becomes `$parameters['-scan']` ŌĆö parameter keys updated to match |
