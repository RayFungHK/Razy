# Razy v0.4.0

**Builds 194–205** | August 2021 – September 2022

The inaugural public release establishing Razy's core architecture: routing, module lifecycle, template engine, database abstraction, and CLI tooling.

---

## Build 194 (2021-08-29)

- Fix `WhereSyntax` SET operator `|=` not working with array values
- Added new event `onDispatch` in Controller — all controllers will be triggered when the route has met
- Minor changes

## Build 195 (2022-07-19)

- Fixed domain with port cannot pass the FQDN and is not matchable

## Build 196 (2022-08-01)

- Code tidy
- Fixed `unpackAsset` throwing error if the asset list is null
- Fixed `unpackAsset` creating null folder if the module has no asset
- Fixed `xcopy` pattern parameter having no effect
- Minor fixes

## Build 197 (2022-08-12)

- Fixed CLI command `set` cannot write the `.htaccess` if the site was created recently

## Build 198 (2022-08-26)

- Support passing parameters to terminal console
- Support STOMP protocol

## Build 199 (2022-08-30)

- Added 2 extra stages — `validate` & `preload` — for better module installation experience
- Added `Controller::goto()` to redirect the page to a specified route within the module
- Added `Controller::landing()` to get the route path by the current URL query (returns empty string if the URL query will not route into the module)

## Build 200 (2022-09-01)

- Added `Block::getTemplate()` and `Entity::getTemplate()` to get the template block object
- Added new parameter `extended_parameter` to allow passing all parameters given by the function tag
- Added `Block::newEntity()` to create an Entity object for processing block structure in template plugins
- Fixed cannot identify the template block within the same level block
- Fixed invalid function tag format causing error

## Build 201 (2022-09-02)

- Template function tag now supports passing arguments into `Template::Plugin` like `{@func:arg1:arg2 param="1" param="2"}`
- `Template::Plugin` now passes a `Razy\Plugin\Container` object into function plugin instead of passing wrapped content and parameters as arguments
- Added `@template` function tag to load preloaded or defined template blocks
- Added `Template::loadTemplate` to load a template file as a global template block
- Renamed `Template::getRootBlock` to `Template::getRoot` (returns `Block` object); `Template::getRoot` returns the `Entity` object

## Build 202 (2022-09-02)

- Fixed function tag `{@if}` nested structure splitting the `{@else}` condition wrongly

## Build 203 (2022-09-09)

- Removed `Terminal\Processor` — parameters storage moved into `Terminal`
- Template variable tag support or-symbol `|` — tries to display the variable in order until one is not empty

## Build 204 (2022-09-21)

- Fixed `Database::WhereSyntax` parsing null syntax not working in some MySQL/MariaDB versions; if either side of expression has `NULL`, the operator converts to `IS` / `IS NOT` instead of `=` / `<>`
- Added `construct()` function to merge one or more arrays recursively by structure
- Added `ipInRange()` function to check if an IP is within a given range

## Build 205 (2022-09-24)

- Fixed undefined error thrown when calling `Distributor::getURLPath()` in CLI mode
- Restored `Terminal::read()` that was deleted accidentally
- Site matching now matches with port first; if no FQDN matched, tries without port — a domain setup without ports allows any connection to match
- Fixed permission of directories created by Razy in `build` command
