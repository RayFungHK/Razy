Logging

Razy provides a PSR-3 compatible logging system with two main classes: Logger (standalone file-based logging) and LogManager (multi-channel logging with pluggable handlers).

Table of Contents

  • Quick Start

  • LogManager (Multi-Channel)
  • Log Handlers
  • Log Levels
  • Message Interpolation

Quick Start

use Razy\Logger;
use Razy\LogManager;
use Razy\LogHandler\FileHandler;

// Simple file-based Logger
$logger = new Logger('/var/log/app');
$logger->info('User logged in', ['user_id' => 42]);

// Multi-channel LogManager
$manager = new LogManager();
$manager->channel('app', new FileHandler('/var/log/app'));
$manager->info('Application started');

Logger (Standalone)

The Logger class writes log entries to files, organized by date.

File-Based Logging

use Razy\Logger;

// Log directory — creates date-based filenames
$logger = new Logger('/var/log/app');
// Creates: /var/log/app/2025-01-15.log

// PSR-3 level methods
$logger->emergency('System is down');
$logger->alert('Action required immediately');
$logger->critical('Critical failure in {component}', ['component' => 'auth']);
$logger->error('Failed to process request');
$logger->warning('Deprecated method called');
$logger->notice('User {user} changed settings', ['user' => 'alice']);
$logger->info('Processing order #{id}', ['id' => 1234]);
$logger->debug('Query executed in {time}ms', ['time' => 42]);

Date-Based Filenames

Log files are automatically named by date:

/var/log/app/2025-01-15.log
/var/log/app/2025-01-16.log

Log entry format:

[2025-01-15 14:30:00] INFO: Processing order #1234

Null-Logger Mode

Pass an empty string to create a no-op logger:

$logger = new Logger('');
$logger->info('This goes nowhere'); // silently ignored

LogManager (Multi-Channel)

The LogManager dispatches log messages to multiple named channels, each backed by a handler.

Named Channels

use Razy\LogManager;
use Razy\LogHandler\FileHandler;
use Razy\LogHandler\StderrHandler;

$manager = new LogManager();

// Register channels with handlers
$manager->channel('app',    new FileHandler('/var/log/app'));
$manager->channel('access', new FileHandler('/var/log/access'));
$manager->channel('error',  new StderrHandler());

// Log to all channels
$manager->info('Request received');     // writes to app, access, error
$manager->error('Something went wrong'); // writes to app, access, error

Channel Override

Register a channel with the same name to replace its handler:

$manager->channel('app', new FileHandler('/var/log/app-v2')); // replaces previous

Inspection

// Check if a channel exists
$manager->hasChannel('app');   // true

// List all channel names
$manager->getChannels();       // ['app', 'access', 'error']

Log Handlers

Handlers implement LogHandlerInterface and determine where log entries go.

Built-In Handlers

FileHandler

Writes to date-based log files:

use Razy\LogHandler\FileHandler;

$handler = new FileHandler('/var/log/app');
// Creates: /var/log/app/2025-01-15.log

StderrHandler

Writes to php://stderr:

use Razy\LogHandler\StderrHandler;

$handler = new StderrHandler();

NullHandler

Discards all log entries:

use Razy\LogHandler\NullHandler;

$handler = new NullHandler();

Custom Handlers

Create a handler by implementing LogHandlerInterface:

use Razy\LogHandler\LogHandlerInterface;

class SlackHandler implements LogHandlerInterface
{
    public function __construct(
        private readonly string $webhookUrl,
    ) {}

    public function handle(string $level, string $message, array $context = []): void
    {
        $payload = json_encode([
            'text' => "[{$level}] {$message}",
        ]);
        // Send to Slack webhook...
    }

    public function flush(): void
    {
        // No-op for immediate sends
    }
}

// Register the custom handler
$manager->channel('slack', new SlackHandler('https://hooks.slack.com/...'));

Log Levels

Follows PSR-3 / RFC 5424 severity levels:

LevelPriorityDescription
debug0Detailed debug information
info1Interesting events
notice2Normal but significant events
warning3Exceptional occurrences that are not errors
error4Runtime errors
critical5Critical conditions
alert6Action must be taken immediately
emergency7System is unusable

Message Interpolation

Use {placeholder} syntax with context arrays:

$logger->info('User {user} purchased {item}', [
    'user' => 'alice',
    'item' => 'Widget',
]);
// Output: [2025-01-15 14:30:00] INFO: User alice purchased Widget

Buffer Mode

Buffer mode collects log entries in memory and writes them all at once when flushed. This is useful for batching I/O operations.

Logger Buffer Mode

$logger = new Logger('/var/log/app');

// Enable buffer mode
$logger->enableBuffer();

$logger->info('Step 1');  // buffered
$logger->info('Step 2');  // buffered
$logger->info('Step 3');  // buffered

// Write all buffered entries at once
$logger->flush();

// Disable buffer mode
$logger->disableBuffer();

LogManager Buffer Mode

$manager = new LogManager();
$manager->channel('app', new FileHandler('/var/log/app'));

// Enable buffer across all channels
$manager->enableBuffer();

$manager->info('Batch operation start');
$manager->info('Processing...');
$manager->info('Batch operation end');

// Flush all channels
$manager->flush();

// Disable buffer mode
$manager->disableBuffer();
