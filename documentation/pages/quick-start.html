Quick Start (5 minutes)

Goal: Get Razy running and execute your first module ??no web server, no domain config, no database. Just PHP + a terminal.

Prerequisites: PHP 8.2+ with ext-zip, ext-curl, ext-json, and Composer.

1. Install & Build

mkdir my-app && cd my-app
composer require rayfunghk/razy
php Razy.phar build
php Razy.phar init dist mysite

You now have a working Razy environment with a distributor called mysite.

2. Create a Module (4 files)

Create these files inside sites/mysite/:

sites/mysite/app/hello/module.php

<?php
return [
    'module_code' => 'app/hello',
    'name'        => 'Hello',
    'author'      => 'Me',
    'version'     => '1.0.0',
];

sites/mysite/app/hello/default/package.php

<?php
return [
    'version' => '1.0.0',
];

sites/mysite/app/hello/default/controller/hello.php

<?php
namespace Razy\Module\hello;

use Razy\Agent;
use Razy\Controller;

return new class extends Controller {
    public function __onInit(Agent $agent): bool
    {
        $agent->addRoute('/', 'index');
        $agent->addRoute('/greet/(:a)', 'greet');
        return true;
    }
};

sites/mysite/app/hello/default/controller/hello.index.php

<?php
return function (): void {
    echo 'Hello, World!' . PHP_EOL;
};

sites/mysite/app/hello/default/controller/hello.greet.php

<?php
return function (string $name): void {
    echo "Hello, {$name}!" . PHP_EOL;
};

3. Run It

Use runapp to test instantly — no web server needed:

php Razy.phar runapp mysite

You'll see the interactive shell prompt:

[mysite]> run /hello/
Hello, World!

[mysite]> run /hello/greet/Razy
Hello, Razy!

[mysite]> routes
# Shows all registered routes

[mysite]> modules
# Shows loaded modules with version info

[mysite]> exit
That's it! You just created a module with two routes and tested it without configuring a web server, domain, or database.

What Just Happened?

FilePurpose
module.phpDeclares the module's identity — module_code must match the directory path
default/package.phpVersion-level configuration — can declare dependencies, API name, Composer requires
hello.phpController — registers routes in __onInit(). Filename matches module name.
hello.index.phpRoute handler — the closure runs when /hello/ is visited
hello.greet.phpRoute handler with parameter — (:a) captures an alphanumeric segment
