YAML

The YAML class provides a pure-PHP YAML parser and emitter with no external dependencies — no ext-yaml or Symfony YAML component required. It exposes 4 static methods for parsing and dumping YAML data.

Methods

MethodReturnDescription
YAML::parse(string $yaml)mixedParse a YAML string into PHP data
YAML::parseFile(string $path)mixedParse a YAML file into PHP data
YAML::dump(mixed $data, int $indent = 2, int $inline = 4)stringEncode PHP data to a YAML string
YAML::dumpFile(string $path, mixed $data, int $indent = 2, int $inline = 4)boolWrite PHP data to a YAML file

Parsing YAML Strings

use Razy\YAML;

$yaml = '
name: MyApp
version: 1.0.0
debug: true
database:
  driver: mysql
  host: 127.0.0.1
  port: 3306
tags:
  - php
  - framework
';

$data = YAML::parse($yaml);
// $data['name']           → "MyApp"
// $data['database']['host'] → "127.0.0.1"
// $data['tags'][0]         → "php"

Parsing YAML Files

use Razy\YAML;

// Parse a YAML configuration file
$config = YAML::parseFile('/path/to/config.yaml');

// Access parsed data
$appName = $config['name'];
$dbHost  = $config['database']['host'];

Dumping to YAML Strings

use Razy\YAML;

$data = [
    'name'    => 'MyApp',
    'version' => '1.0.0',
    'features' => [
        'routing'   => true,
        'templates' => true,
        'database'  => false,
    ],
];

// Default: 2-space indent, inline at depth 4
$yamlString = YAML::dump($data);

// Custom indent and inline depth
$yamlString = YAML::dump($data, 4, 2);

The $indent parameter controls the number of spaces per indentation level. The $inline parameter sets the depth at which values are rendered inline (on a single line).

Writing YAML Files

use Razy\YAML;

$settings = [
    'app' => [
        'name'  => 'MyApp',
        'debug' => false,
    ],
    'database' => [
        'driver' => 'mysql',
        'host'   => '127.0.0.1',
    ],
];

// Write to file, returns true on success
$success = YAML::dumpFile('/path/to/output.yaml', $settings);

Example: Module Manifests

use Razy\YAML;

// Parse a module's manifest file
$manifest = YAML::parseFile('sites/mysite/blog/default/manifest.yaml');

echo $manifest['module_code'];   // "blog"
echo $manifest['version'];       // "1.2.0"
echo $manifest['author'];        // "Ray Fung"

// List required dependencies
foreach ($manifest['require'] as $dep => $version) {
    echo "Requires $dep $version\n";
}

Example: Data Serialization

use Razy\YAML;

// Export application state to YAML
$state = [
    'generated_at' => date('Y-m-d H:i:s'),
    'modules' => [
        ['name' => 'auth',  'status' => 'active'],
        ['name' => 'blog',  'status' => 'active'],
        ['name' => 'cache', 'status' => 'disabled'],
    ],
    'stats' => [
        'requests' => 15420,
        'errors'   => 3,
    ],
];

// Save state snapshot
YAML::dumpFile('/snapshots/state.yaml', $state);

// Re-load state later
$restored = YAML::parseFile('/snapshots/state.yaml');
